## 第3页：设计目标与原则

### 设计目标
**[设计目标示意图：业务目标与技术目标平衡]**

- **业务目标**：
  - 为在校大学生提供统一的多模型AI对话平台
  - 简化AI模型使用流程，提升学习效率
  - 支持主流大模型服务的快速接入

- **技术目标**：
  - 构建高可用、可扩展的Web应用架构
  - 实现模块化设计，便于维护和升级
  - 确保系统安全性和数据隐私保护

### 设计原则
**[设计原则图示：核心原则展示]**

| 原则类别 | 具体内容 | 实现方式 |
|---------|---------|---------|
| **高内聚低耦合** | 模块职责单一，依赖清晰 | 分层架构，接口隔离 |
| **可扩展性** | 支持新模型快速接入 | 插件化设计，配置驱动 |
| **可维护性** | 代码清晰，文档完整 | 统一编码规范，自动化测试 |
| **安全性** | 数据保护，权限控制 | 加密存储，JWT认证 |
| **性能优化** | 响应快速，资源高效 | 缓存策略，异步处理 |

### 主要数据表结构
**[数据表结构示意图：字段类型和约束]**

#### 用户表 (users)
```sql
CREATE TABLE users (
    user_id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(255),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login DATETIME,
    is_active BOOLEAN DEFAULT TRUE
);
```

#### 对话表 (conversations)
```sql
CREATE TABLE conversations (
    conversation_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    title VARCHAR(200),
    model_used VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
```

```http
DELETE /api/v1/conversations/{id} # 删除对话
POST   /api/v1/conversations/{id}/messages # 发送消息
```

### 典型调用时序图
**[用户登录时序图：完整的认证流程]**

```
用户登录时序：
1. 前端 → 后端: POST /login (用户名密码)
2. 后端 → 数据库: 验证用户信息
3. 数据库 → 后端: 返回用户数据
4. 后端: 生成JWT令牌
5. 后端 → 前端: 返回令牌和用户信息
6. 前端: 存储令牌，跳转到主页面
```
